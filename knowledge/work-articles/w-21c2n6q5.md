## Objective

Close the open lifecycle state across the five merged snapshot follow-up work articles, cancel the benchmark spike with a justified reason, cut `3.0.0-alpha.5` for the surface that shipped, and update the research note's follow-up table with the final phases. Also fix the CLI gap that made this cleanup harder than it should have been: `pnpm exec tsx src/bin.ts work advance` did not accept `--reason` or a `skip-guard` bypass, so closing work articles with a real audit trail required writing one-off tsx scripts.

## Motivation

After the five follow-ups (#60 #61 #62 #63 #64) landed on main, the work articles still sit in `review` phase because the `all_reviewers_approved` guard blocks done, and there is no interactive reviewer for these PRs (they were merged directly by the session author). Similarly, `w-uvp3azdf` (bench spike) cannot close as `done` — the exit criteria require real results which require an API-budgeted session. The right move is to cancel with an audit-recorded reason rather than leaving the article hanging in `enrichment`.

Finally, the whole surface (`environment_snapshot` tools + Dolt persistence + guard + dashboard band + agent docs) is large enough to warrant a named release cut. All CHANGELOG entries still live under `[Unreleased]`.
