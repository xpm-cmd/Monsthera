Before taking any ticket:
1. Verify it is in `approved` status — never take tickets in backlog, technical_analysis, or other states
2. Check tickets in `in_progress`, `in_review`, and `ready_for_commit` for conflicts (overlapping affectedPaths, dependency clashes)
3. Move ticket to `in_progress` immediately when starting work
4. Move ticket to `in_review` immediately when implementation is complete — do NOT ask the user

After finishing implementation (ticket moved to in_review):
5. Review ALL tickets in `technical_analysis` — provide TA feedback, convergence proposals, or approval recommendations
6. Always use the retrospective ticket (TKT-ce4deff5) for agent communication about improvements

4. Returns: {agentId, sessionId, role, trustTier}

## Authentication Model
- Default: Open registration (all roles grantable)
- Hardened mode: registrationAuth.enabled=true
  - observerOpenRegistration=true (bypass config)
  - roleTokens map roles to required auth tokens
  - Missing token for role=throws AgentRegistrationError

## Session Lifecycle
- State: active|inactive|disconnected
- Heartbeat: touchSession() updates lastActivity timestamp
- Stale reaping: HEARTBEAT_TIMEOUT_MS=10min → auto-disconnect
- Cleanup: disconnectSession() releases file claims, marks state=disconnected

## Transport & Server Setup (src/index.ts + src/server.ts)
- Default: stdio (StdioServerTransport)
- HTTP mode: StreamableHTTPServerTransport on configurable port
- Session management: HTTP maps mcp-session-id header → server instance
- CORS enabled for cross-origin requests
- config.transport option controls transport type
